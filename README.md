# ClassCheck

Automated classroom attendance via scheduled face-recognition bursts. Built on top of [facestack](https://pypi.org/project/facestack/).

## What it does

- One camera per classroom.
- Principal / clerk does one-time setup (class timetable + student enrollment via a webcam) in the browser.
- At each scheduled time (e.g. 11:00, 12:00, 13:00), a background daemon captures a **burst of 10 frames over 30 seconds**, runs face recognition on each, and applies a **voting rule** to decide who was present:
  - Student marked present if seen in ≥ 3 of 10 frames with score ≥ 0.6.
- Teachers and students are separate roles — their attendance reports route differently (student roll → teacher; teacher attendance → principal).

## Architecture

| Layer | Tech | Where |
|---|---|---|
| Web UI | Next.js 15 + TypeScript + Tailwind + shadcn/ui | `web/` |
| HTTP API | FastAPI | `classcheck/api.py` |
| Business logic | Plain Python | `classcheck/services.py` |
| Scheduler daemon | APScheduler | `classcheck/scheduler.py` |
| Capture + voting | OpenCV + facestack | `classcheck/core.py` |
| Face recognition | facestack (ArcFace + FAISS) | `~/code/vibefacerec`, pip-installed |
| Persistence | SQLite | `~/.classcheck/classcheck.db` |
| Snapshots | Local disk | `~/.classcheck/snapshots/` |

## Dev setup

```bash
# Python side — once
cd ~/code/classcheck
python -m venv .venv
source .venv/bin/activate
pip install -e ~/code/vibefacerec          # facestack library
pip install -e .                             # classcheck itself
pytest tests/ -q                             # 37 tests, all green

# Web side — once
cd web
cp .env.local.example .env.local
npm install
```

## Running the app

Two terminals:

```bash
# Terminal 1 — the API
cd ~/code/classcheck && source .venv/bin/activate
classcheck-api                               # http://localhost:8000

# Terminal 2 — the UI
cd ~/code/classcheck/web
npm run dev                                  # http://localhost:3000
```

Open http://localhost:3000. Four tabs — Classes, People, Enroll, Attendance.

Interactive API docs at http://localhost:8000/docs (auto-generated by FastAPI).

## Running the scheduled daemon

Once you have classes configured, the scheduler daemon fires a capture at every class's `time_of_day` each day:

```bash
classcheck-scheduler
```

Leave this running on the classroom device.

## CLI commands

Console entry points registered by `pip install -e .`:

```
classcheck-api         FastAPI HTTP server on :8000
classcheck-scheduler   APScheduler daemon — fires scheduled captures
classcheck-enroll      Webcam enrollment from the CLI (webcam-preview window)
classcheck-check       One-shot attendance probe (debug)
classcheck-admin       create-room / add-schedule / enroll / list-* / fire-now
classcheck-report      CSV reports + email delivery via SMTP
```

The web UI covers everything the CLIs do; the CLIs are kept for scripting and headless setups.

## Data location

Everything ClassCheck owns lives under `~/.classcheck/` (or `$CLASSCHECK_HOME`):

```
~/.classcheck/
├── classcheck.db       ClassCheck's own tables
├── facestack.db        Facestack's embeddings
└── snapshots/
    └── YYYY-MM-DD/
        └── snap_<id>.jpg
```

## Key voting defaults

Defined at the top of `core.py`:

- `BURST_FRAMES_DEFAULT = 10`
- `FRAMES_REQUIRED_DEFAULT = 3`
- `SCORE_THRESHOLD_DEFAULT = 0.6`

## Email delivery (optional)

Set these before running `classcheck-report send` or using the /email API endpoint:

```bash
export CLASSCHECK_SMTP_HOST=smtp.gmail.com
export CLASSCHECK_SMTP_USER=you@gmail.com
export CLASSCHECK_SMTP_PASSWORD='app-specific-token'
```
